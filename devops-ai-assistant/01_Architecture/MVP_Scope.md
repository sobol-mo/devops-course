## 1. Визначення MVP

Цей MVP описує мінімальний набір функцій, необхідних для побудови робочого AI-агента, який:

- Запускається на **власному сервері** (локальна VM у цьому курсі, реальний VPS у production)
- Спілкується через **Telegram**
- Виконує базові задачі персональної продуктивності
- Демонструє всі ключові **DevOps-практики**, що вивчаються в курсі

### Ключова гіпотеза

> «AI-агент з автоматизованим розгортанням може надавати надійну,
> конфіденційну допомогу у щоденних завданнях — і бути повністю відновлений із коду менш ніж за 30 хвилин.»

### Критерії успіху MVP

| Критерій              | Метрика                                                                  |
| --------------------- | ------------------------------------------------------------------------ |
| **Доступність**       | Агент відповідає на Telegram-повідомлення протягом 5 секунд, 24/7.      |
| **Цілісність даних**  | Всі дані користувача зберігаються у self-hosted PostgreSQL базі даних.   |
| **Суверенітет даних** | Всі дані знаходяться на сервері (VM), що контролюється користувачем.     |
| **Відтворюваність**   | Повне середовище відновлюється з коду менш ніж за 30 хвилин.            |
| **Автоматизація**     | Нульова ручна конфігурація сервера — всі налаштування через Ansible.     |

## 2. Функціонал у межах scope

### 2.1 Основний агент (OpenClaw)

| Функція                 | Опис                                                          | Пріоритет |
| ----------------------- | ------------------------------------------------------------- | --------- |
| **Telegram-інтерфейс**  | Отримання та відправка повідомлень через Telegram Bot.        | Must      |
| **Нагадування**         | «Нагадай мені подзвонити Івану завтра о 10:00»                | Must      |
| **Нотатки**             | Зберегти думку або ідею для подальшого пошуку.                | Must      |
| **Календар (Google)**   | Перевірити зайнятість, створити подію в Google Calendar.      | Should    |
| **Веб-підсумок**        | Завантажити URL і підсумувати вміст сторінки.                 | Should    |
| **Email (Gmail)**       | Читати та надсилати листи через природньомовні команди.       | Could     |

### 2.2 Постійне сховище (PostgreSQL)

| Функція                   | Опис                                                         | Пріоритет |
| ------------------------- | ------------------------------------------------------------ | --------- |
| **Persistent Memory**     | Зберігати уподобання та контекст між розмовами.             | Must      |
| **Фінансовий трекінг**   | Записувати доходи/витрати через природньомовні команди.      | Could     |

> **Примітка щодо PostgreSQL:** База даних запускається як Docker-контейнер на VM.
> Це **обов'язковий** компонент архітектури, не опціональний.
> Студенти розгорнуть і підключаться до неї в рамках Модуля 8.

## 4. Технічна архітектура

**Стек:** Node.js • OpenClaw framework • PostgreSQL (Docker) • Ansible • systemd

### Чому OpenClaw (а не власний Python-бот)?

OpenClaw — це open-source фреймворк для AI-агентів. Його використання дозволяє студентам:

- **Швидко отримати результат:** Робочий Telegram-бот з інструментами готовий із першого дня.
- **Зосередитись на DevOps:** Студенти конфігурують, розгортають та автоматизують реальний проєкт — а не будують його з нуля.
- **Вивчити реальну кодову базу:** Структура проєкту демонструє професійні патерни.

### Ключові архітектурні рішення

| Рішення                              | Обґрунтування                                                          |
| ------------------------------------- | ---------------------------------------------------------------------- |
| **Окремий системний користувач**      | Користувач `openclaw` без `sudo`. Обмежує наслідки можливого злому.    |
| **Прив'язка тільки до localhost**     | Агент слухає на `127.0.0.1`. Зовнішній доступ — виключно через SSH-тунель. |
| **Infrastructure as Code**            | Ansible playbooks. Сервер відновлюється з коду, а не з пам'яті.       |
| **systemd-сервіс**                    | Автозапуск при старті системи та автоперезапуск при збою. Логи через `journalctl`. |
| **PostgreSQL у Docker**               | Розгортається однією командою `docker-compose up`. Ізольовано та відтворювано. |
| **Локальна VM замість реального VPS** | Без витрат для студентів. Навички повністю переносяться на реальну хмару. |

## 5. Мінімізація ризиків

| Ризик                      | Мінімізація                                                                    |
| -------------------------- | ------------------------------------------------------------------------------ |
| **Збій VM**                | Infrastructure as Code (Ansible) дозволяє повне відновлення менш ніж за 30 хв. |
| **Порушення безпеки**      | Principle of Least Privilege — користувач `openclaw` без системних прав.       |
| **Витік токенів**          | API-токени зберігаються у `.env` файлі, ніколи не потрапляють у Git.          |
| **Вартість для студентів** | Вся інфраструктура запускається локально — жодних платних хмарних сервісів.    |

## 6. Визначення «Готово» (Definition of Done)

MVP вважається завершеним, коли студент може продемонструвати:

- [ ] OpenClaw-агент відповідає на Telegram-повідомлення з **локальної VM**.
- [ ] Конфігурація сервера **повністю автоматизована** через Ansible (playbook відтворює все з нуля на свіжій VM).
- [ ] Агент керується **`systemd`-сервісом** (автозапуск при старті, автоперезапуск при збою).
- [ ] SSH-тунель є **єдиною точкою доступу** до UI агента (відкритий лише порт 22).
- [ ] PostgreSQL запущено як **Docker-контейнер** і агент зберігає дані між перезапусками.
- [ ] **CI/CD pipeline** автоматично розгортає зміни з гілки `main` на VM.
